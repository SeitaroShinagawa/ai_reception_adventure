---

### ユーザー
「攻略AI学会懇親会」というタイトルのゲームを作ろうと思っています。まずは計画について相談に乗ってもらえますか。

想定しているものとしては
・AI学会の懇親会で新しい知り合いを作るのを目的としたゲーム。最終的にAI学会で学生がどのように知り合いを作っていけばよいかを学べる教育的要素のあるゲームにしたい
・公開して、ブラウザ上で誰でも遊べるのが望ましい
・選択肢が提示されるシナリオゲームが良い

### AI
面白いコンセプトですね！教育的要素のある選択肢型シナリオゲームということで、いくつか設計の方向性を提案します。

#### ゲームの構造案

**シナリオの流れ**
- プレイヤーは学生として懇親会会場に到着
- 複数のNPC（教授、企業の人、他大学の学生など）が存在
- 各キャラクターとの会話で「良い選択肢」「悪い選択肢」を選ぶ
- 最終的に「知り合いスコア」や「名刺交換数」などで結果が変わる

**教育的要素の組み込み方**
- 悪い選択をした場合にフィードバックを表示（「なぜその選択が良くなかったか」）
- エンディング後に「攻略のコツ」をまとめて表示
- リアルなシチュエーション（自己紹介のタイミング、話題の選び方など）を反映

**技術スタック候補**

| 選択肢 | 特徴 |
|--------|------|
| **Twine** | 選択肢ゲーム特化、HTML出力でそのまま公開可能、学習コスト低 |
| **React + JSON** | 柔軟性高い、GitHub Pagesで無料公開可能、開発コスト中 |
| **Ren'Py** | ビジュアルノベル特化、Web出力可能、立ち絵が映える |

**確認事項**
1. キャラクターのビジュアルはどうするか？（立ち絵あり・テキストのみ・アイコン程度）
2. シナリオの長さは？（短編10〜15分 or 複数ルートで30分以上）
3. AIとのリアルタイム会話は想定するか、固定シナリオか？

#### アーキテクチャ案

**シナリオデータ構造（JSON）**
```
scenes/
  intro.json        # 会場到着・チュートリアル
  characters/
    professor.json  # 教授との会話シナリオ
    student.json    # 他大学学生
    engineer.json   # 企業エンジニア
    ...
  ending.json       # 結果・フィードバック
```

**確率的要素の実装イメージ**
- 懇親会の「時間帯」（前半・中盤・後半）ごとに出会えるキャラクターをプールで管理
- 各キャラクターに出現確率を設定し、プレイのたびに異なる組み合わせに
- 例：「教授は必ず1人出る、残り2枠はランダム」という設計

**画面構成案**
```
[背景：懇親会会場]
[キャラクター立ち絵]
[名前・所属テキスト]
[会話テキストボックス]
[選択肢ボタン × 2〜3個]
[スコア表示：名刺交換数など]
```

**実装ステップ案**
1. 土台作り - 画面レイアウト、シナリオエンジン（JSON読み込み・選択肢分岐）
2. キャラクター設計 - 登場人物の種類・立ち絵・性格設定
3. シナリオ執筆 - 各キャラクターの会話ツリー
4. 確率システム - 出現ロジック実装
5. エンディング - スコア集計・フィードバック表示
6. 公開 - GitHub Pages or Vercel

**実装内容：**
- `src/types/index.ts` - 型定義（Character, Scene, Choice, GameState）

- `src/engine/useGameEngine.ts` - シナリオ進行・スコア管理のReact hook
- `src/components/SceneView.tsx` - メインゲーム画面
- `src/components/FeedbackModal.tsx` - 選択後のフィードバックモーダル
- `src/components/EndingView.tsx` - エンディング・スコア・攻略コツ表示
- `src/App.tsx` - アプリ本体
